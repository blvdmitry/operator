The InputCheckbox also has an `inderminate` state.
The indeterminate state is used when multiple sub-options are grouped under a parent option. The indeterminate state will represent that multiple sub-options (but not all of them) are selected in the list.

### Controlled and uncontrolled

All input components in BUI support controlled and uncontrolled version.
Uncontrolled version is the default behavior where you can pass it a `defaultChecked` and the browser will control the InputCheckbox's checked or unchecked state.

<Example name="uncontrolled" />

Controlled version is enabled once you pass the `checked` prop to InputCheckbox .
This means that clicking on the InputCheckbox will trigger `onChange` but you will have to update the `checked` prop using your own state.

<Example name="controlled" />

## Accessibility

- The component is built to be keyboard accessible
- When an error is displayed for a not checked required checkbox hide the error when it gets checked
- Errors should have `alert` role to be announced when being applied
- Try to avoid using `required` checkboxes
- When grouped - checkboxes should be wrapped with a `fieldset` and provide a descriptive `legend` for screen readers
- When implementing `required` checkboxes try to add the `required` attribute, but at least set the `aria-required` to `true` to make sure that the input is announced as such
