  one to two handles that can be dragged over a bar to select a numeric value.

## Usage

```jsx
import { InputSlider } from "@bookingcom/bui-react";
import type { InputSliderProps } from "@bookingcom/bui-react";
```

By default, InputSlider works as a form field for a single value selection.
Because of that, `name` propery is required and you have to make sure to either pass a `label` or an `ariaLabel` to make it accessible for the users.
You can subscribe to the changes using `onChange` event handler which will give access to the new value.

<Example name="default" />

Additionally, InputSlider supports a `range` flag for working with a value range selection.
In this case, `onChange` will return `minValue` and `maxValue` arguments instead of `value`.

<Example name="range" />

In situations when the range in between min and max boundaries is too because, user experience can be improved by using a custom `interval` to skip intermidiate values.

<Example name="interval" />

### Value Visibility

Use `valueVisibility` to control the way value and tooltips are rendered. It accepts three string values: hidden, caption and tooltip.

- caption: value is rendered, tooltips are visible on hover
- tooltip: value is hidden, tooltips are always visible
- hidden: value and tooltips are always hidden

By default, `caption` is applied. This way value is always rendered as a range of numbers; you can modify that behaviour with `renderValue` property.
This also allows you to render the value using custom translation tags.

<Example name="valueRender" />

Same applies for the values rendered in the tooltips.
For example, instead of rendering a number in the tooltip, you might also want to display a currency sign:

<Example name="tooltipRender" />

It's possible to not render the value as a separate text element.
In that case tooltips will always be shown instead.

You can hide both value and tooltips by passing `hidden`.

## Accessibility

- The user can manipulate the component handles using the keyboard, using the arrow keys after focusing on the element. The default number that will be increased/decreased of the value when the user presses an arrow is `1`, and this value can be changed setting the `interval` property.
- Screen readers will ignore the tooltips, since they will read the selected values directly from the `input[range]` element.
